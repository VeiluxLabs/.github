## Overview

**VEILUX** is a privacy-first, AI-native blockchain platform engineered for confidential applications, decentralized intelligence, and institutional-grade infrastructure.

At its center is **Photon** — a deliberately lightweight protocol kernel. Around it, VEILUX layers cryptographic privacy, Byzantine fault-tolerant consensus, and a modular extension system, giving organizations a foundation to coordinate, transact, and compute securely while retaining full control over who sees what.

We build for a future where **privacy, intelligence, and verifiability coexist without compromise.**

## Why VEILUX

| Challenge | The VEILUX Approach |
|-----------|---------------------|
| Public ledgers expose every transaction | Confidential by default — data is disclosed to stakeholders only |
| Monolithic chains are heavy and rigid | A featherweight core; every capability is an opt-in module |
| Regulators need oversight, businesses need secrecy | Cryptographic selective disclosure — a "glass box" for auditors, privacy from competitors |
| AI workloads need verifiable, on-chain logic | AI-native execution with verifiable commitments |

## The Platform

### Photon — The Core Kernel

Photon is the minimal execution foundation of VEILUX. It does little by design, and that is its strength.

- Deterministic state transitions
- Content-addressed, authenticated state
- Cryptographic commitments and Merkle integrity
- Event-driven, modular execution
- A tiny dependency surface for long-term security and maintainability

### Veil — The Confidentiality Layer

Veil delivers the **VeilLedger** privacy model: one logically shared ledger where no participant sees data they are not entitled to.

- Stakeholder-specific transaction visibility
- Per-party encrypted views
- A single, globally verifiable commitment history
- Scoped selective disclosure for auditors and regulators

Participants can verify that something happened — and that it was valid — without ever seeing the underlying confidential data.

### Aurora — The Consensus Engine

Aurora is VEILUX's stake-weighted Byzantine fault-tolerant consensus.

- Deterministic, stake-aware proposer selection
- Two-phase voting with 2/3+ finality
- Equivocation detection and validator accountability
- Live multi-node finality with a synchronized, tamper-evident chain

### Prisms — Modular Capabilities

Prisms are protocol extensions that add functionality without inflating the core. Each Prism is self-contained and composes through a cascading execution pipeline.

| Prism | Capability |
|-------|------------|
| **AI** | On-chain model registry and verifiable inference, with optional local model execution |
| **Storage** | Content-addressed data storage with reference-counted retention |
| **Token** | Fungible digital assets |
| **NFT** | Non-fungible assets and collections |
| **Contract** | Deterministic smart-contract execution |

Future ecosystem modules may include identity, account abstraction, data availability, zero-knowledge services, oracle networks, and enterprise integrations.

## Design Principles

**Privacy by Default**
Information should be disclosed intentionally, never exposed by architecture.

**Verifiability Without Disclosure**
Outcomes must be provable to all participants without revealing confidential data.

**Modular Evolution**
Capabilities evolve independently from consensus, so the protocol can grow without growing fragile.

**Sustainable Simplicity**
A lightweight core delivers stronger security, greater flexibility, and lasting maintainability.

## For Developers

VEILUX is written in **Rust** and organized around a modular, extensible architecture that prioritizes correctness, performance, and operational simplicity.

- Build custom **Prisms** to introduce new functionality without forking the core
- Leverage built-in privacy, consensus, and persistence
- Run a single persistent node or a multi-validator network out of the box
- Ship to Linux, Windows, and macOS via automated, reproducible release pipelines

Whether you are building AI services, financial infrastructure, identity systems, or application-specific networks, VEILUX provides a flexible foundation for innovation.

## Vision

Digital infrastructure is converging on three demands at once: it must be **private**, it must be **intelligent**, and it must be **verifiable**. Today these goals are usually traded off against one another.

VEILUX exists to remove that trade-off — and to provide the foundation for a more confidential, more intelligent digital economy.
